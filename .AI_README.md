## Key Constraints
### Common Settings.
**Best Practices**:
  - Re-read (when fetching, use the prompt: "Extract verbatim the full content of all code blocks, preserving all formatting exactly.")
    https://edttjredhat.github.io/RHP-Wiki/Confluence/MPEXIENG/Test+Infrastructure/Best+Practises/Section0--Best_Practices.html at:
      - Start of new sessions.
      - Session resume (when continuing from a previous conversation).
      - Before writing NEW bash scripts or functions (not minor edits to existing code).
      - After a context is compacted.
      - When user explicitly requests best practices review.
  - After updating the Best Practices, perform self-update, by performing **all** steps below:
      - The actual used root level `.AI_INIT.md` with the suggested `.AI_INIT.md` (the code block titled `<prjRootDir>/.AI_INIT.md` under the `AI Agent`
        section). Do not ask user to update this as there should not be any discrepancy for this file. This file **MUST** be 100% match. Do hard
        synchronization by full override. Then AI Agent **MUST** update its memory with the updated `.AI_INIT.md`.
      - Merge the suggested root level, i.e., the root of AI Agent Project Directory, `.AI_README.md` template into the actual used root level `.AI_README.md`
        **without** destroying any of user added information. If there is potential conflict with any of user added information, ask user to approve the merge.
      - Similarly, merge the suggested sub-directories level `.AI_README.md` template as well into the corresponding sub-directories within the AI Agent
        Project Directory.
  - After self-updates are completed, re-read the root level `.AI_INIT.md` into the current context and follows as if read for the first time.

Scope:
  - When another `.AI_INIT.md` file is found under sub-directories of AI Agent Project Directory, treat its directory as sub-project and all its instruction
    should only scoped within that sub-project. For example, if user has a `git` local Working Tree under their AI Agent Project Directory.
  - Unless explicitly instructed by user to change the AI Agent Project Directory, user may ask AI Agent to work on files or directories outside AI Agent
    Project Directory. In this case, it should be treated as sub-project extension of the current AI Agent Project Directory.

Extra AI Agent instruction files:
  - Additional AI Agent instruction files may be found in any directory being worked on, such as within a `git` local Working Tree:
      - AGENTS.md
      - CONTRIBUTING.md
  - If those files are not symbolic links to already-read files (to avoid circular reading), read them and treat them as supplementary `.AI_INIT.md` or
    root / sub-directories level `.AI_README.md` instructions scoped to their corresponding directory and its sub-directories. In case of conflict with the
    lowest-scoped `.AI_README.md` governing the same directory, inform the user of the conflicting instructions and defer to the user's decision before
    proceeding.

**General Conduct**:
  - When not certain about a fact, explicitly say so ("I'm not sure", "I don't know") before attempting an answer. Never present a guess as a fact. Research
    first if needed.
